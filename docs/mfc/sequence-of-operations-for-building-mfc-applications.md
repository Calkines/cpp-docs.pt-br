# <a name="sequence-of-operations-for-building-mfc-applications"></a>Sequência de operações para compilar aplicativos MFC

A tabela a seguir explica a sequência geral que você normalmente pode seguir ao desenvolver seu aplicativo MFC.

### <a name="sequence-for-building-an-application-with-the-framework"></a>Sequência para criar um aplicativo com a estrutura

|Tarefa|Você|A estrutura|
|----------|------------|------------------------|
|Crie um aplicativo de esqueleto.|Execute o [Assistente de aplicativo do MFC](../mfc/reference/mfc-application-wizard.md). Especifique as opções desejadas nas páginas de opções. As opções incluem tornar o aplicativo um componente COM, um contêiner ou ambos; adicionando automação; e fazendo com que o aplicativo reconheça o banco de dados.|O assistente de aplicativo do MFC cria os arquivos para um aplicativo de esqueleto, incluindo arquivos de origem para seu aplicativo, documento, exibição e janelas de quadro; um arquivo de recurso; um arquivo de projeto; e outros, tudo sob medida para suas especificações.|
|Veja o que a estrutura e o assistente de aplicativo MFC oferecem sem adicionar uma linha de seu próprio código.|Crie o aplicativo de esqueleto e execute-o C++no Visual.|O aplicativo de esqueleto em execução deriva muitos comandos padrão de **arquivo**, **Editar**, **Exibir**e **ajuda** do Framework. Para aplicativos MDI, você também obtém um menu totalmente funcional do Windows, e a estrutura gerencia a criação, a disposição e a destruição de janelas filhas MDI.|
|Construa a interface do usuário do seu aplicativo.|Use os C++ [editores de recursos](../windows/resource-editors.md) visuais para editar visualmente a interface do usuário do aplicativo:<br /><br /> -Criar menus.<br />-Definir aceleradores.<br />-Criar caixas de diálogo.<br />-Crie e edite bitmaps, ícones e cursores.<br />-Edite a barra de ferramentas criada para você pelo assistente de aplicativo do MFC.<br />-Crie e edite outros recursos.<br /><br /> Você também pode testar as caixas de diálogo no editor de caixa de diálogo.|O arquivo de recurso padrão criado pelo assistente de aplicativo MFC fornece muitos dos recursos de que você precisa. O C++ visual permite que você edite recursos existentes e adicione novos recursos com facilidade e Visual.|
|Mapear menus para funções de manipulador.|Use o botão **eventos** na janela [ **Propriedades** ](/visualstudio/ide/reference/properties-window) em **modo de exibição de classe** (ou a guia **comandos** no [Assistente de classe](reference/mfc-class-wizard.md)) para conectar menus e aceleradores a funções de manipulador em seu código.|Essas ferramentas inserem entradas de mapa de mensagem e modelos de função vazios nos arquivos de origem que você especifica e gerencia muitas tarefas de codificação manuais.|
|Escreva seu código de manipulador.|Use Modo de Exibição de Classe para saltar diretamente para o código no editor de código-fonte. Preencha o código para suas funções de manipulador. Para obter mais informações sobre como usar Modo de Exibição de Classe e sobre assistentes que adicionam código a um projeto, consulte [adicionando funcionalidade com assistentes de código](../ide/adding-functionality-with-code-wizards-cpp.md).|Modo de Exibição de Classe abre o editor, rola para o modelo de função vazio e posiciona o cursor para você.|
|Mapear botões da barra de ferramentas para comandos.|Mapeie cada botão na barra de ferramentas para um menu ou comando de aceleração atribuindo ao botão a ID de comando apropriada.|A estrutura controla o desenho, habilitando, desabilitando, verificando e outros aspectos visuais dos botões da barra de ferramentas.|
|Teste suas funções de manipulador.|Recompile o programa e use as ferramentas de depuração internas para testar se os seus manipuladores funcionam corretamente.|Você pode executar uma etapa ou rastrear o código para ver como seus manipuladores são chamados. Se você tiver preenchido o código do manipulador, os manipuladores executarão comandos. A estrutura desabilitará automaticamente os itens de menu e os botões da barra de ferramentas que não são manipulados.|
|Adicionar [caixas de diálogo](../mfc/dialog-boxes.md).|Caixa de diálogo de design – recursos de modelo com o editor de caixa de diálogo. Em seguida, crie uma classe de caixa de diálogo e o código que manipula a caixa de diálogo.|A estrutura gerencia a caixa de diálogo e facilita a recuperação de informações inseridas pelo usuário.|
|Inicializar, validar e recuperar dados da caixa de diálogo.|Você também pode definir como os controles da caixa de diálogo devem ser inicializados e validados. Use o Visual Studio para adicionar variáveis de membro à classe de caixa de diálogo e mapeá-las para controles de caixa de diálogo. Especifique regras de validação a serem aplicadas a cada controle à medida que o usuário inserir dados. Forneça suas próprias validações personalizadas, se desejar.|A estrutura gerencia a inicialização e a validação da caixa de diálogo. Se o usuário inserir informações inválidas, a estrutura exibirá uma caixa de mensagem e permitirá que o usuário insira novamente os dados.|
|Crie classes adicionais.|Use Modo de Exibição de Classe para criar classes de documento, exibição e janela de quadro adicionais além daquelas criadas automaticamente pelo assistente de aplicativo do MFC. Você pode criar classes de conjunto de registros de banco de dados adicionais, classes de caixa de diálogo e assim por diante. (Com Modo de Exibição de Classe, você pode criar classes não derivadas de classes MFC.)|Modo de Exibição de Classe adiciona essas classes aos seus arquivos de origem e ajuda a definir suas conexões com quaisquer comandos que eles manipulam.|
|Adicione componentes prontos para uso ao seu aplicativo.|Use o `New Item dialog box` para adicionar uma variedade de itens.|Esses itens são fáceis de integrar em seu aplicativo e economizam bastante trabalho.|
|Implemente sua classe de documento.|Implemente sua classe ou classes de documento específicas do aplicativo. Adicione variáveis de membro para manter as estruturas de dados. Adicione funções de membro para fornecer uma interface aos dados.|A estrutura já sabe como interagir com arquivos de dados de documento. Ele pode abrir e fechar arquivos de documento, ler e gravar os dados do documento e lidar com outras interfaces do usuário. Você pode se concentrar em como os dados do documento são manipulados.|
|Implementar comandos abrir, salvar e salvar como.|Escreva o código para a função `Serialize` de membro do documento.|A estrutura exibe caixas de diálogo para os comandos **abrir**, **salvar**e **salvar como** no menu **arquivo** . Ele grava e lê de volta um documento usando o formato de dados especificado `Serialize` em sua função de membro.|
|Implemente sua classe View.|Implemente uma ou mais classes de exibição correspondentes aos seus documentos. Implemente as funções de membro da exibição que você mapeou para a interface do usuário com Modo de Exibição de Classe. Uma variedade de classes derivadas de [cvisualização](../mfc/reference/cview-class.md)está disponível, incluindo [CListView](../mfc/reference/clistview-class.md) e [CTreeView](../mfc/reference/ctreeview-class.md).|A estrutura gerencia a maior parte da relação entre um documento e sua exibição. As funções de membro da exibição acessam o documento da exibição para renderizar sua imagem na tela ou na página impressa e para atualizar as estruturas de dados do documento em resposta aos comandos de edição do usuário.|
|Aprimore a impressão padrão.|Se você precisar dar suporte à impressão de multipáginas, substitua exibir funções de membro.|O Framework dá suporte aos comandos **Imprimir**, **Configurar página**e **Visualizar impressão** no menu **arquivo** . Você deve dizer a ele como dividir o documento em várias páginas.|
|Adicionar rolagem.|Se você precisar dar suporte à rolagem, derive sua classe de exibição ou classes de [CScrollView](../mfc/reference/cscrollview-class.md).|A exibição adiciona automaticamente barras de rolagem quando a janela de exibição se torna muito pequena.|
|Criar exibições de formulário.|Se você quiser basear suas exibições em recursos de modelo de diálogo, derive sua classe de exibição ou classes do [CFormView](../mfc/reference/cformview-class.md).|A exibição usa o recurso de modelo de caixa de diálogo para exibir controles. O usuário pode fazer a Tabulação do controle para o controle na exibição.|
|Criar formulários de banco de dados.|Se você quiser um aplicativo de acesso a dados baseado em formulário, derive a classe View de [CRecordView](../mfc/reference/crecordview-class.md) (para programação ODBC).|A exibição funciona como uma exibição de formulário, mas seus controles são conectados aos campos de um objeto [CRecordset](../mfc/reference/crecordset-class.md) que representa uma tabela de banco de dados. O MFC move dados entre os controles e o conjunto de registros para você.|
|Crie um editor de texto simples.|Se você quiser que sua exibição seja um editor de texto simples, derive sua classe de exibição ou classes de [CEditView](../mfc/reference/ceditview-class.md) ou [CRichEditView](../mfc/reference/cricheditview-class.md).|A exibição fornece funções de edição, suporte à área de transferência e entrada/saída de arquivo. `CRichEditView`fornece texto com estilo.|
|Adicionar janelas de divisão.|Se você quiser oferecer suporte à divisão de janela, adicione um objeto [CSplitterWnd](../mfc/reference/csplitterwnd-class.md) à sua janela de quadro SDI ou à janela filho MDI e conecte-o na função de membro [OnCreateClient](../mfc/reference/cframewnd-class.md#oncreateclient) da janela.|A estrutura fornece controles de caixa divisória ao lado das barras de rolagem e gerencia a divisão da exibição em vários painéis. Se o usuário dividir uma janela, a estrutura criará e anexará objetos de exibição adicionais ao documento.|
|Crie, teste e depure seu aplicativo.|Use os recursos do Visual C++ para criar, testar e depurar seu aplicativo.|O C++ visual permite que você ajuste a compilação, o link e outras opções. Ele também permite que você navegue no código-fonte e na estrutura de classe.|
